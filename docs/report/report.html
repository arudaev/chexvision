Abstract

This project tackles automated chest X-ray pathology detection at scale using the NIH ChestX-ray14 dataset (112,120 frontal-view radiographs, 14 disease labels). We implement and compare two deep learning approaches under identical evaluation conditions: a custom residual CNN trained entirely from scratch with Squeeze-and-Excitation channel attention, and a DenseNet-121 fine-tuned by transfer learning from ImageNet, following the CheXNet design. Both models share a single dual-head output that solves two tasks in one forward pass: a 14-class multi-label classification task (detect every pathology present) and a binary screening task (Normal vs Abnormal). On the held-out validation split, the DenseNet model reaches a macro AUC-ROC of 0.846, slightly exceeding the published CheXNet benchmark of 0.841, while the from-scratch CNN reaches 0.801 with no pretrained weights. We report per-class performance, analyse where each model wins and fails, and discuss the data, modelling, and infrastructure limitations that bound these results. All heavy compute runs on free cloud GPUs, with a fully reproducible, tested, and continuously integrated codebase.

1. Problem Statement

Chest radiography is the most common diagnostic imaging exam in the world, yet expert interpretation is slow, expensive, and unevenly available. A model that can flag likely pathologies and triage normal from abnormal films would help radiologists prioritise their reading lists and extend basic screening to settings without specialist coverage. This is the clinical motivation behind the NIH ChestX-ray14 release and the landmark CheXNet result, which demonstrated radiologist-level pneumonia detection on this exact data.

The course brief requires two models on the same dataset, with one model implemented and trained from scratch and a second model built with an alternative approach (transfer learning), and an explicit, quantitative comparison between them. We frame the problem as two complementary supervised tasks on the same images:

  • Task 1 - Multi-label classification. For each image, predict an independent probability for every one of the 14 pathologies. Labels are not mutually exclusive: a single film can show Effusion, Infiltration, and Atelectasis at once. This is the harder, clinically richer task and our primary evaluation target.
  • Task 2 - Binary classification. Predict a single Normal vs Abnormal score (any finding present). This is the screening / triage task and a useful sanity signal on the shared features.

Both tasks are solved by one network with two output heads on a shared backbone, so a single forward pass produces both predictions. This keeps the comparison between the two architectures clean: only the backbone changes, while the task formulation, loss, data, and evaluation are held fixed.

Primary metric: macro-averaged AUC-ROC. AUC-ROC is threshold-independent and robust to the severe class imbalance in this dataset, which makes it the standard choice for ChestX-ray14 and the metric reported by CheXNet. Accuracy would be misleading here - a model that predicts "absent" for the rarest pathology (Hernia, well under 1% prevalence) would still score above 99% accuracy on that class while being clinically useless.

2. Dataset Description

We use NIH ChestX-ray14 (Wang et al., 2017), one of the largest publicly available labelled chest X-ray collections: 112,120 frontal-view images from 30,805 unique patients, each tagged with zero or more of 14 pathology labels mined from radiology reports. The 14 labels are Atelectasis, Cardiomegaly, Effusion, Infiltration, Mass, Nodule, Pneumonia, Pneumothorax, Consolidation, Edema, Emphysema, Fibrosis, Pleural Thickening, and Hernia. Images with none of these are labelled "No Finding".

2.2 Class imbalance - the central data challenge

The dataset is severely imbalanced in two ways. First, roughly 54% of all images carry no finding at all, so the binary task is close to balanced (about 46% abnormal) but the multi-label task is dominated by negatives. Second, prevalence across the 14 pathologies spans more than two orders of magnitude: Infiltration and Effusion are common, while Hernia appears in well under 1% of films. Left unaddressed, a network minimises its loss most easily by predicting "absent" for every rare class.

We address this directly with per-class positive weighting in the loss (Section 3.2). For each pathology we compute pos_weight = negatives / positives on the training split and pass it to a numerically stable BCEWithLogitsLoss. For the rarest class this drives the positive weight to roughly 500, so a single missed Hernia costs the network about as much as 500 missed Effusions. This is the single most important design choice for getting non-trivial performance on the rare tail.

2.3 Big-data engineering: a 320 x 320 pre-resized dataset

The raw NIH release is about 45 GB of 1024 x 1024 PNGs - too large to stream efficiently from free cloud GPUs within a time budget. We therefore built a one-time preprocessing pipeline (a Kaggle kernel) that decodes the raw archives, resizes every image to 320 x 320 RGB, and writes 36 Parquet shards totalling about 7.97 GB, published as a versioned Hugging Face dataset (arudaev/chest-xray-14-320). The dataset revision hash is pinned in source, so every training run reads byte-identical data. We chose 320 x 320 rather than the common 224 px baseline because chest pathology often hinges on small, low-contrast findings (nodules, fine infiltrates); the higher resolution preserves that detail, at the cost of a smaller batch size to fit GPU memory.

3. Methodology

3.1 Dual-head, single-backbone design

Both models share an identical output design: a shared backbone extracts a feature vector once, and two small linear heads read from it - a multi-label head (14 logits, one independent sigmoid per pathology) and a binary head (1 logit, Normal vs Abnormal). This is deliberate. Computing both tasks from shared features is cheaper than two separate networks, the binary task acts as an auxiliary signal that regularises the shared representation, and - critically for the report - it isolates the variable under study. When we compare "scratch" against "DenseNet", only the backbone differs; the heads, loss, data, and evaluation are held constant.

3.2 Training methodology (shared by both models)

Every design choice below is implemented in raw PyTorch (no high-level training framework) so that each decision is explicit and defensible.

ComponentChoice and justification
LossCombined BCEWithLogitsLoss: 1.0 x multilabel + 0.5 x binary. Fused sigmoid + BCE for numerical stability. The binary task is down-weighted because it is easier and should support, not dominate, the harder multi-label objective.
Class balancingPer-class pos_weight on the multi-label loss, computed from training-split prevalence. Directly counteracts the long-tailed label distribution (Section 2.2). The binary head uses plain BCE because that task is already near-balanced.
OptimizerAdamW (decoupled weight decay 1e-4). Adam-family convergence with proper L2 regularisation; the default choice for modern vision fine-tuning.
LR scheduleCosine annealing over the full epoch budget, with a short warm-up. Smoothly decays the learning rate to near zero for a stable final solution.
Mixed precision (AMP)fp16 forward/backward with fp32 master weights. About 1.7x faster on the T4/P100 GPUs and roughly halves memory, which is what makes 320 px training feasible in the free 9-hour Kaggle window.
Gradient accumulationPhysical batch 24 accumulated over 4 steps = effective batch 96. Recovers the stability of a large batch despite the memory cost of 320 px images.
Gradient clippingGlobal-norm clip at 1.0, to keep the weighted, imbalanced loss from producing occasional exploding updates on rare-class batches.
ReproducibilityA single seed (42) is set across Python, NumPy, PyTorch, and CUDA, with deterministic cuDNN. Pinned dataset revision + frozen split + embedded config = repeatable runs.

3.3 Medically motivated data augmentation

Augmentation is chosen to reflect real radiographic variation rather than generic image noise. Each transform has a clinical rationale:

AugmentationWhy it is anatomically valid
Random horizontal flipChest anatomy is broadly bilateral; a mirrored film is still a plausible chest X-ray.
Random rotation (+/- 15 deg)Patients are rarely perfectly aligned to the detector.
Affine translate / shearSimulates positioning shifts and beam-angle variation.
Color jitter (brightness/contrast)Compensates for differing X-ray exposure settings between machines.
Gaussian blur (p = 0.2)Models varying sharpness from patient motion or detector resolution.
Random erasing (p = 0.1)Mimics radio-opaque artefacts (leads, clips, implants) and forces distributed, not single-region, features.
ImageNet normalizationStandard for the ImageNet-pretrained DenseNet; applied to both models so inputs are identical.
Two optional enhancements - CLAHE and label smoothing. The codebase also implements Contrast Limited Adaptive Histogram Equalisation (local contrast enhancement in LAB colour space, to expose low-contrast findings) and label smoothing (epsilon = 0.1, to regularise against the dataset's noisy report-mined labels). Both are config-driven. As reported transparently in Section 6.4, our final DenseNet checkpoint was trained with both enabled, while the from-scratch checkpoint that produced the reported numbers predates enabling them. We flag this rather than hide it; closing that gap is listed in future work (Section 8).

4. Model 1 - Custom CNN From Scratch

Model 1 is a residual convolutional network built and trained entirely from scratch - no pretrained weights of any kind. The architecture follows ResNet-50-equivalent depth and adds Squeeze-and-Excitation channel attention. It has roughly 23 million parameters, all of which are learned on chest X-rays alone.

4.1 Architecture walk-through

  • Stem. 7x7 convolution (stride 2) + batch norm + ReLU + 3x3 max-pool. Captures low-level edges and textures and immediately reduces spatial resolution by 4x for efficiency.
  • Four residual stages, depths [3, 4, 6, 3]. This is ResNet-50 depth. We deliberately chose it over a lighter [2, 2, 2, 2] (ResNet-18) baseline: detecting 14 simultaneous, often co-occurring pathology signals needs more representational capacity, and the extra depth (about 23M vs 11M parameters) is the capacity we judged worth its training cost.
  • Residual blocks. Two 3x3 convolutions with a skip connection. Residual connections are what make a network this deep trainable at all - they give gradients a short path and mitigate vanishing gradients. Batch norm on every conv stabilises training and permits a higher learning rate.
  • Squeeze-and-Excitation (SE) attention. After each block, an SE module global-average-pools the feature map to a per-channel descriptor, learns per-channel importance weights through a small two-layer bottleneck (sigmoid-gated), and rescales the channels. This is channel-wise feature selection.
  • Global average pooling + dropout (0.5). Pooling collapses spatial dimensions with no learned parameters (far less overfitting-prone than a large flatten + dense layer); heavy dropout further regularises the high-capacity model.
  • Dual heads. Linear 512 -> 14 (multi-label) and Linear 512 -> 1 (binary).
  • Initialization. Kaiming (He) initialization for the ReLU conv stack; Xavier for the linear heads. Correct variance scaling is essential when training a deep network from random weights.
Why SE attention, specifically for this problem. Different pathologies activate different feature channels - the texture that signals Emphysema is not the texture that signals Cardiomegaly. SE lets the network amplify the disease-relevant channels and suppress background-texture channels on a per-image basis, which is exactly the kind of competition that multi-label medical imaging creates. It adds negligible parameters and is, in effect, learned disease-specific feature gating (Hu et al., 2018).

4.2 What "from scratch" costs and buys

Training 23M parameters from random initialization on a noisy, imbalanced medical dataset is genuinely hard: the network must learn edges, textures, and anatomy before it can learn disease, and it gets no head start from natural-image pretraining. The payoff is a model whose every weight is explainable from first principles and a clean baseline for measuring exactly how much ImageNet transfer is worth (Section 7). As the results show, it reaches a respectable 0.801 macro AUC-ROC - clearly learning real pathology signal - while landing below the transfer-learning model, which is the expected and instructive outcome.

5. Model 2 - DenseNet-121 Transfer Learning

Model 2 is a DenseNet-121 pretrained on ImageNet and fine-tuned for our two tasks - the alternative approach required by the brief, and a direct nod to CheXNet (Rajpurkar et al., 2017), which established exactly this backbone as the reference architecture for ChestX-ray14. Only about 7.9 million of its parameters sit in the trainable feature/head path on top of the dense backbone.

5.1 Why DenseNet-121 and transfer learning

  • Dense connectivity. Every layer receives the feature maps of all preceding layers in its block. This maximises feature reuse, strengthens gradient flow, and reaches strong accuracy with far fewer parameters than a comparable ResNet - a good fit for a free-GPU budget.
  • ImageNet pretraining transfers. Low-level features - edges, gradients, blob and texture detectors - are largely domain-agnostic. Starting from them means the network spends its training budget adapting to radiographic structure rather than relearning vision from zero. This is precisely why it converges faster and higher than Model 1.
  • Proven on this exact data. CheXNet reported 0.841 macro AUC-ROC with a single-head DenseNet-121; adopting it gives us a credible, literature-anchored target.

5.2 Two-phase fine-tuning strategy

Fine-tuning a pretrained backbone with freshly initialised heads is delicate: large early gradients from the random heads can wash out ("catastrophically forget") the valuable ImageNet features. We avoid this with a two-phase schedule.

  • Phase 1 (epochs 1-5): backbone frozen. Only the feature layer and the two heads train, at the higher learning rate. The heads learn to read the fixed ImageNet features without disturbing them.
  • Phase 2 (epoch 6 onward): full unfreeze. All layers become trainable at a 10x lower learning rate, so the backbone can gently specialise its mid- and high-level features to chest-X-ray textures without destroying what pretraining provided.

This schedule is the main reason the model converges so quickly: its best validation checkpoint lands at epoch 18, far earlier than the from-scratch model's epoch 60, because it begins from a strong initialization rather than random noise.

6.5 Inference-time enhancements

The evaluation and demo paths add two zero-training-cost techniques, implemented and available in the deployed system:

  • Test-time augmentation (TTA). Average the sigmoid probabilities over 4 views (original, horizontal flip, and +/- 7 degree rotations). This reduces prediction variance at no training cost.
  • Two-model ensemble. Average the TTA probabilities of both models. Because a from-scratch CNN and a pretrained DenseNet have different inductive biases, they fail on different images, so averaging their outputs is a classic, reliable way to gain a little macro AUC over either model alone.

The headline finding. Transfer learning beats from-scratch training by about 4.5 macro AUC-ROC points while converging more than 3x faster and training fewer parameters. This is the textbook transfer-learning result, and seeing it cleanly reproduced on a 112k-image medical dataset - under identical heads, loss, and evaluation - is exactly the controlled comparison the project set out to make. The from-scratch model is not a failure: at 0.801 macro AUC it has clearly learned genuine pathology signal, and it serves its purpose as the baseline that quantifies what ImageNet pretraining is worth here.

7.1 Where they agree and disagree

Both models follow the same per-class ordering - both find Edema, Cardiomegaly, and Emphysema easy and Infiltration and Pneumonia hard - which tells us the difficulty ranking is a property of the data (label noise and finding subtlety), not of either architecture. Where they differ is the ceiling: the DenseNet lifts the whole curve, most visibly on the mid-difficulty findings (Mass, Pneumothorax, Atelectasis) where richer pretrained features help separate subtle structure from background. Because their residual errors are not identical, the ensemble in Section 6.5 can recover a little of each model's unique correct predictions.

8. Discussion and Limitations

8.1 Limitations

  • Label noise. ChestX-ray14 labels are mined from free-text radiology reports, not verified lesion annotations. They are patient/report-level and known to be noisy, which places a hard ceiling on achievable AUC - especially for diffuse findings like Infiltration and Pneumonia, our two weakest classes.
  • Validation, not test, numbers. The figures reported here are on the validation split used for checkpoint selection. The 22k-image test split is held out; final unbiased test numbers should be computed before any external claim, and will typically sit slightly below validation.
  • Unequal preprocessing between the two reported runs. As stated in Section 6.4, the scratch checkpoint was trained without CLAHE and label smoothing while the DenseNet checkpoint used both. The architectural conclusion is robust, but the precise margin is not a perfectly controlled ablation.
  • No external validation. Both models are trained and evaluated on a single institution's distribution. Performance is expected to degrade on images from different equipment, populations, or preprocessing - a well-documented generalisation gap for chest-X-ray models.
  • Not for clinical use. This is an educational system. Its predictions must not substitute for professional medical judgement.

8.2 Future work

  • Re-train the from-scratch CNN with CLAHE and label smoothing enabled, for a fully controlled scratch-vs-transfer ablation.
  • Report final held-out test metrics and per-class confidence intervals.
  • Add Grad-CAM saliency overlays (implemented in the visualization module) to a sample of films for qualitative, clinically-checkable interpretability.
  • Quantify the ensemble's macro-AUC gain numerically and tune per-class decision thresholds for the binary screening use case.

9. Conclusion

We built and compared two deep learning models for chest X-ray pathology detection on the 112,120-image NIH ChestX-ray14 dataset, each solving a 14-class multi-label task and a binary screening task through one shared backbone with two heads. A custom SE-ResNet trained entirely from scratch reached 0.801 macro AUC-ROC, and a DenseNet-121 fine-tuned by transfer learning reached 0.846, slightly surpassing the CheXNet reference while converging more than three times faster. The controlled comparison cleanly demonstrates the value of transfer learning on large-scale medical imaging, while the from-scratch baseline grounds that value in a concrete number. The per-class results align with clinical intuition and with known label-noise limits of the dataset, and the whole system is reproducible, tested, and deployed as a live demo.
